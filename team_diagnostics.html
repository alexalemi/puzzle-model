Data Overview

Model

Extends Model 2r with logsumexp team alpha (rates add in time domain) + Amdahl’s law (serial fraction) + per-bucket corrections:

alpha_i(t) = alpha_i + (delta_0 + delta_i) · t    (per-member velocity)
alpha_parallel = −mB · log(Σ exp(−alpha_i(t) / mB))
amdahl = mB · log(1 + s·(K−1))    (serial overhead)

alpha_eff = alpha_parallel + amdahl + η[bucket(K)]
mean = mu + alpha_eff + beta[puzzle] + piece_correction + practice
log_time ~ StudentT(nu, mean, sigma)

For solo (K=1): logsumexp = alpha_i + velocity, amdahl = 0, η = 0 — exactly Model 2r. Four new parameters beyond 2r:

  • s — serial fraction (0 = fully parallel, 1 = serial)
  • η2, η3, η4+ — per-bucket residual corrections

Velocity is applied per-member inside the logsumexp. Unregistered team members map to a shared __unknown__ puzzler (shrinks toward alpha=0, i.e. average ability).

Team Parameters

The serial fraction s from Amdahl’s law determines how much of the work can be parallelized (s=0 fully parallel, s=1 serial). Per-bucket η corrections capture deviations from the Amdahl prediction.

Diagnostic: Solo Model vs Team Reality

What happens if we predict team times using the solo-only model with logsumexp of member alphas? If team members contribute independent rates (1/time), the total rate is the sum, giving αeff = −1000·log10(Σ 10−αi/1000). Points on the diagonal mean the parallel-rates model fits perfectly.
